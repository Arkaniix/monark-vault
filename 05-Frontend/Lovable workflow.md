# Lovable workflow

Déploiement frontend via Lovable, connecté au repo GitHub `Arkaniix/monark-market`.

## Workflow dev
1. Claude produit spec détaillée → Etienne valide
2. Claude génère prompt Claude Code VPS (avec STOP points) + prompt Lovable séparé
3. Backend d'abord (Claude Code), puis frontend (Lovable)
4. Éditions chirurgicales > réécriture complète

## Prompts en attente
- Repair Guide frontend
- `ReportDialog` diagnostics UI
